📊 Normalization vs Standardization

Master Feature Scaling for Machine Learning

🎯Why Feature Scaling Matters

The Problem:

Imagine you're comparing people's age (20-80 years) with their salary ($30,000-$200,000). The salary values will dominate any distance calculations because of their large magnitude!

🏠 Real Estate Example:

Without Scaling:

  • House Size: 1,500 sq ft
  • Bedrooms: 3
  • Price: $450,000

The price would completely overshadow the bedroom count in any ML algorithm!

⚖️Normalization vs Standardization

🔧 Normalization (Min-Max Scaling)

X_scaled = (X - X_min) / (X_max - X_min)

Range: [0, 1]

When to use:

  • ✅ When you know the min/max bounds
  • ✅ Image processing (pixel values)
  • ✅ Neural networks expecting [0,1] inputs

Effect: Squeezes all values into 0-1 range

📈 Standardization (Z-score)

Z = (X - μ) / σ

Mean: 0, Std Dev: 1

When to use:

  • ✅ Data follows Gaussian distribution
  • ✅ Unknown bounds or outliers present
  • ✅ Most ML algorithms

Effect: Centers data around 0 with unit variance

💻Python Implementation

Normalization Code

from sklearn.preprocessing import MinMaxScaler
import pandas as pd

# Read the dataset
df = pd.read_csv('winequality_red.csv', sep=';')

# Select features to scale
features_to_scale = ['alcohol', 'sulphates']

# Create MinMaxScaler
scaler = MinMaxScaler()

# Fit and transform
df[features_to_scale] = scaler.fit_transform(df[features_to_scale])

# Values now between 0 and 1!
print(df.head())

Standardization Code

from sklearn.preprocessing import StandardScaler
import pandas as pd

# Read the dataset
df = pd.read_csv('winequality_red.csv', sep=';')

# Select features to scale
features_to_scale = ['alcohol', 'sulphates']

# Create StandardScaler
scaler = StandardScaler()

# Fit and transform
df[features_to_scale] = scaler.fit_transform(df[features_to_scale])

# Mean = 0, Std Dev = 1!
print(df.head())

⚖️When to Use Each Method

Normalization (Min-Max)

✅ Pros

  • Preserves zero values
  • Bounded range [0,1]
  • Good for images/pixels
  • Intuitive interpretation

❌ Cons

  • Sensitive to outliers
  • New data might fall outside [0,1]
  • Not good if distribution matters

Standardization (Z-score)

✅ Pros

  • Less affected by outliers
  • Works well with Gaussian data
  • Maintains shape of distribution
  • Most algorithms prefer this

❌ Cons

  • Unbounded range
  • Doesn't preserve zero values
  • Less intuitive values

🌍Real-World Applications

🖼️ Image Processing

Use Normalization: Pixel values (0-255) → (0-1)

CNNs and image classifiers expect normalized pixel values for faster convergence.

💰 Credit Scoring

Use Standardization: Income, Age, Credit History have vastly different scales.

Standardization ensures no single feature dominates the model.

🏥 Medical Diagnosis

Use Standardization: Blood pressure (120/80), Temperature (98.6°F), Heart rate (60-100 bpm)

Different medical measurements need standardization for fair comparison.

📱 Recommender Systems

Use Normalization: User ratings (1-5 stars) → (0-1)

Normalized ratings help in collaborative filtering algorithms.

📌Key Takeaways

Remember:

  • 📊 Normalization: Squeezes to [0,1] - Good for bounded data
  • 📈 Standardization: Mean=0, StdDev=1 - Good for unbounded data
  • 🎯 General Rule: When in doubt, use Standardization!
  • 🌲 Tree-based models: Don't need any scaling
  • 🔍 Distance-based models: Always need scaling
