The Problem:

‚öñÔ∏èNormalization vs Standardization

üîß Normalization (Min-Max Scaling)

X_scaled = (X - X_min) / (X_max - X_min)

Range: [0, 1]

When to use:

  • ‚úÖ When you know the min/max bounds
  • ‚úÖ Image processing (pixel values)
  • ‚úÖ Neural networks expecting [0,1] inputs

Effect: Squeezes all values into 0-1 range

üìà Standardization (Z-score)

Z = (X - Œº) / œÉ

Mean: 0, Std Dev: 1

When to use:

  • ‚úÖ Data follows Gaussian distribution
  • ‚úÖ Unknown bounds or outliers present
  • ‚úÖ Most ML algorithms

Effect: Centers data around 0 with unit variance

üíªPython Implementation

Normalization Code

from sklearn.preprocessing import MinMaxScaler
import pandas as pd

# Read the dataset
df = pd.read_csv('winequality_red.csv', sep=';')

# Select features to scale
features_to_scale = ['alcohol', 'sulphates']

# Create MinMaxScaler
scaler = MinMaxScaler()

# Fit and transform
df[features_to_scale] = scaler.fit_transform(df[features_to_scale])

# Values now between 0 and 1!
print(df.head())

Standardization Code

from sklearn.preprocessing import StandardScaler
import pandas as pd

# Read the dataset
df = pd.read_csv('winequality_red.csv', sep=';')

# Select features to scale
features_to_scale = ['alcohol', 'sulphates']

# Create StandardScaler
scaler = StandardScaler()

# Fit and transform
df[features_to_scale] = scaler.fit_transform(df[features_to_scale])

# Mean = 0, Std Dev = 1!
print(df.head())

‚öñÔ∏èWhen to Use Each Method

Normalization (Min-Max)

‚úÖ Pros

  • Preserves zero values
  • Bounded range [0,1]
  • Good for images/pixels
  • Intuitive interpretation

‚ùå Cons

  • Sensitive to outliers
  • New data might fall outside [0,1]
  • Not good if distribution matters

Standardization (Z-score)

‚úÖ Pros

  • Less affected by outliers
  • Works well with Gaussian data
  • Maintains shape of distribution
  • Most algorithms prefer this

‚ùå Cons

  • Unbounded range
  • Doesn't preserve zero values
  • Less intuitive values

üåçReal-World Applications

üñºÔ∏è Image Processing

Use Normalization: Pixel values (0-255) ‚Üí (0-1)

CNNs and image classifiers expect normalized pixel values for faster convergence.

üí∞ Credit Scoring

Use Standardization: Income, Age, Credit History have vastly different scales.

Standardization ensures no single feature dominates the model.

üè• Medical Diagnosis

Use Standardization: Blood pressure (120/80), Temperature (98.6¬∞F), Heart rate (60-100 bpm)

Different medical measurements need standardization for fair comparison.

üì± Recommender Systems

Use Normalization: User ratings (1-5 stars) ‚Üí (0-1)

Normalized ratings help in collaborative filtering algorithms.

üìåKey Takeaways

Remember:

  • üìä Normalization: Squeezes to [0,1] - Good for bounded data
  • üìà Standardization: Mean=0, StdDev=1 - Good for unbounded data
  • üéØ General Rule: When in doubt, use Standardization!
  • üå≤ Tree-based models: Don't need any scaling
  • üîç Distance-based models: Always need scaling
