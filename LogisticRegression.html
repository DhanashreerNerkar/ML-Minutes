üìä What is Logistic Regression?

Logistic Regression is a statistical method for predicting binary outcomes (yes/no, true/false, 0/1) by modeling the probability that an instance belongs to a particular class. Despite its name, it's a classification algorithm, not regression!

P(y=1|x) = œÉ(z) = 1 / (1 + e^(-z))
where z = Œ≤‚ÇÄ + Œ≤‚ÇÅx‚ÇÅ + Œ≤‚ÇÇx‚ÇÇ + ... + Œ≤‚Çôx‚Çô

üéØ Purpose

Predict the probability of binary outcomes (0 or 1, Yes or No, True or False)

üìä Output Range

Always between 0 and 1 (probability), unlike linear regression (-‚àû to +‚àû)

üîÑ Transform Function

Uses sigmoid/logistic function to map linear combination to probabilities

üé≠ Decision Boundary

Creates a linear boundary in feature space (non-linear in probability space)

‚öñÔ∏è Threshold

Default 0.5, but can be adjusted based on business needs (precision vs recall)

üîç Interpretability

Coefficients represent log-odds change for unit increase in feature

üé® The Sigmoid Function

üìà Properties

  • S-shaped curve
  • Output always between 0 and 1
  • œÉ(0) = 0.5 (midpoint)
  • Differentiable everywhere
  • œÉ'(z) = œÉ(z)(1 - œÉ(z))

üî¢ Interpretation

  • z > 0 ‚Üí P > 0.5 ‚Üí Predict Class 1
  • z < 0 ‚Üí P < 0.5 ‚Üí Predict Class 0
  • z = 0 ‚Üí P = 0.5 ‚Üí Decision boundary
  • |z| large ‚Üí High confidence
  • |z| small ‚Üí Low confidence

‚úÖ Advantages

  • Probabilistic predictions
  • No tuning required
  • Fast training and prediction
  • Works well for linearly separable data
  • Interpretable coefficients
  • Less prone to overfitting (with regularization)
  • Can be extended to multiclass

‚ùå Limitations

  • Assumes linear decision boundary
  • Sensitive to outliers
  • Requires more data for stable results
  • Can't solve non-linear problems directly
  • Assumes independence of features
  • Sensitive to multicollinearity

üßÆ Mathematical Foundation

Odds and Log-Odds

Odds = P(success) / P(failure) = p / (1-p)

Log-Odds (Logit) = ln(odds) = ln(p/(1-p)) = Œ≤‚ÇÄ + Œ≤‚ÇÅx‚ÇÅ + ... + Œ≤‚Çôx‚Çô

üìä From Linear to Logistic

Linear Model: y = Œ≤‚ÇÄ + Œ≤‚ÇÅx

Apply Sigmoid: P(y=1) = 1/(1 + e^(-(Œ≤‚ÇÄ + Œ≤‚ÇÅx)))

Result: Probabilities between 0 and 1

üéØ Maximum Likelihood Estimation

Find Œ≤ that maximizes:

L(Œ≤) = ‚àè P(y·µ¢|x·µ¢,Œ≤)

No closed-form solution ‚Üí Use gradient descent

üìâ Cost Function (Log Loss)

J(Œ≤) = -1/m Œ£[y¬∑log(≈∑) + (1-y)¬∑log(1-≈∑)]

Convex function ‚Üí Global minimum exists

# Logistic Regression Mathematics Implementation import numpy as np import matplotlib.pyplot as plt class LogisticRegressionMath: """ Logistic Regression from scratch with mathematical details """ def __init__(self, learning_rate=0.01, iterations=1000): self.lr = learning_rate self.iterations = iterations self.costs = [] def sigmoid(self, z): """Sigmoid activation function""" return 1 / (1 + np.exp(-np.clip(z, -500, 500))) # Clip to prevent overflow def sigmoid_derivative(self, z): """Derivative of sigmoid function""" s = self.sigmoid(z) return s * (1 - s) def cost_function(self, X, y, theta): """ Binary cross-entropy loss / Log loss J(Œ∏) = -1/m * Œ£[y*log(h(x)) + (1-y)*log(1-h(x))] """ m = len(y) z = X @ theta h = self.sigmoid(z) # Add small epsilon to prevent log(0) epsilon = 1e-7 h = np.clip(h, epsilon, 1 - epsilon) cost = -1/m * (y @ np.log(h) + (1-y) @ np.log(1-h)) return cost def gradient(self, X, y, theta): """ Gradient of cost function ‚àÇJ/‚àÇŒ∏ = 1/m * X^T * (h(x) - y) """ m = len(y) z = X @ theta h = self.sigmoid(z) gradient = 1/m * X.T @ (h - y) return gradient def fit(self, X, y): """Train using gradient descent""" # Add bias term X = np.column_stack([np.ones(len(X)), X]) m, n = X.shape # Initialize parameters self.theta = np.zeros(n) # Gradient descent for i in range(self.iterations): # Calculate cost cost = self.cost_function(X, y, self.theta) self.costs.append(cost) # Calculate gradient grad = self.gradient(X, y, self.theta) # Update parameters self.theta -= self.lr * grad # Print progress if i % 100 == 0: print(f"Iteration {i}, Cost: {cost:.4f}") return self def predict_proba(self, X): """Predict probabilities""" X = np.column_stack([np.ones(len(X)), X]) return self.sigmoid(X @ self.theta) def predict(self, X, threshold=0.5): """Predict classes""" return (self.predict_proba(X) >= threshold).astype(int) def decision_boundary(self, X): """ Calculate decision boundary For 2D: x2 = -(Œ∏0 + Œ∏1*x1) / Œ∏2 """ if len(self.theta) == 3: # 2D case x1 = np.linspace(X[:, 0].min(), X[:, 0].max(), 100) x2 = -(self.theta[0] + self.theta[1] * x1) / self.theta[2] return x1, x2 else: raise ValueError("Decision boundary only for 2D features") # Demonstration np.random.seed(42) # Generate synthetic data n_samples = 200 X_class0 = np.random.randn(n_samples//2, 2) + [-2, -2] X_class1 = np.random.randn(n_samples//2, 2) + [2, 2] X = np.vstack([X_class0, X_class1]) y = np.hstack([np.zeros(n_samples//2), np.ones(n_samples//2)]) # Train model model = LogisticRegressionMath(learning_rate=0.1, iterations=1000) model.fit(X, y) # Visualize results fig, axes = plt.subplots(1, 3, figsize=(15, 5)) # 1. Data and Decision Boundary axes[0].scatter(X_class0[:, 0], X_class0[:, 1], c='red', label='Class 0', alpha=0.6) axes[0].scatter(X_class1[:, 0], X_class1[:, 1], c='blue', label='Class 1', alpha=0.6) # Plot decision boundary x1_boundary, x2_boundary = model.decision_boundary(X) axes[0].plot(x1_boundary, x2_boundary, 'g--', linewidth=2, label='Decision Boundary') axes[0].set_xlabel('Feature 1') axes[0].set_ylabel('Feature 2') axes[0].set_title('Logistic Regression Decision Boundary') axes[0].legend() axes[0].grid(True, alpha=0.3) # 2. Cost Function Over Iterations axes[1].plot(model.costs) axes[1].set_xlabel('Iteration') axes[1].set_ylabel('Cost (Log Loss)') axes[1].set_title('Cost Function Convergence') axes[1].grid(True, alpha=0.3) # 3. Sigmoid Function z = np.linspace(-10, 10, 100) sigmoid_values = model.sigmoid(z) axes[2].plot(z, sigmoid_values, linewidth=2, color='purple') axes[2].axhline(y=0.5, color='r', linestyle='--', alpha=0.5) axes[2].axvline(x=0, color='gray', linestyle='--', alpha=0.5) axes[2].set_xlabel('z (linear combination)') axes[2].set_ylabel('œÉ(z) (probability)') axes[2].set_title('Sigmoid Activation Function') axes[2].grid(True, alpha=0.3) plt.tight_layout() plt.show() print(f"\nModel Parameters:") print(f"Intercept (Œ≤‚ÇÄ): {model.theta[0]:.4f}") print(f"Coefficients (Œ≤‚ÇÅ, Œ≤‚ÇÇ): {model.theta[1:]}") print(f"Final Cost: {model.costs[-1]:.4f}")

Œ∏‚±º := Œ∏‚±º - Œ± ¬∑ (1/m) ¬∑ Œ£(h(x‚Å±) - y‚Å±) ¬∑ x‚±º‚Å±

where h(x) = œÉ(Œ∏·µÄx) is the hypothesis function

üîÑ Regularization in Logistic Regression

L2 Regularization (Ridge)

Cost: J(Œ∏) + ŒªŒ£Œ∏‚±º¬≤

Effect: Shrinks coefficients

Use: Multicollinearity, overfitting

L1 Regularization (Lasso)

Cost: J(Œ∏) + ŒªŒ£|Œ∏‚±º|

Effect: Feature selection

Use: Sparse features

Elastic Net

Cost: J(Œ∏) + Œª‚ÇÅŒ£|Œ∏‚±º| + Œª‚ÇÇŒ£Œ∏‚±º¬≤

Effect: Balance of L1 and L2

Use: Many correlated features

üìä Model Evaluation Metrics

üéØ Accuracy

Formula: (TP + TN) / Total

Use: Balanced datasets

Limitation: Misleading for imbalanced data

üîç Precision

Formula: TP / (TP + FP)

Meaning: Of predicted positives, how many correct?

Focus: Minimize false positives

üì¢ Recall (Sensitivity)

Formula: TP / (TP + FN)

Meaning: Of actual positives, how many found?

Focus: Minimize false negatives

‚öñÔ∏è F1-Score

Formula: 2 √ó (Precision √ó Recall) / (Precision + Recall)

Use: Balance precision and recall

Best for: Imbalanced datasets

üìà ROC-AUC

ROC: TPR vs FPR curve

AUC: Area under ROC curve

Range: 0.5 (random) to 1.0 (perfect)

üìâ Log Loss

Formula: -Œ£[y¬∑log(≈∑) + (1-y)¬∑log(1-≈∑)]

Use: Probabilistic evaluation

Lower is better: 0 is perfect

Metric Best When Example Use Case
Accuracy Classes balanced General classification
Precision False positives costly Spam detection, fraud detection
Recall False negatives costly Disease diagnosis, security threats
F1-Score Need balance Information retrieval
ROC-AUC Ranking important Credit scoring, risk assessment
