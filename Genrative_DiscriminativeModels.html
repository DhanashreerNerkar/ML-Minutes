üöÄ Classic Examples

üìß Generative: Naive Bayes for Spam Detection

How it works:

  1. Learns word distributions in spam emails: P(words|spam)
  2. Learns word distributions in ham emails: P(words|ham)
  3. Learns overall spam probability: P(spam)
  4. Can generate "typical" spam or ham emails
P(spam|email) = P(email|spam) √ó P(spam) / P(email)

üìà Discriminative: Logistic Regression

How it works:

  1. Directly models P(spam|email features)
  2. Learns weights for each feature
  3. Finds optimal decision boundary
  4. Cannot generate new emails
P(Y=1|X) = 1 / (1 + e^(-w^T x))

üéØ When to Use Which?

Use Generative When:

  • Need to generate new samples
  • Want to detect anomalies/outliers
  • Have missing data to impute
  • Need to understand data distribution
  • Want model interpretability
  • Building creative AI applications

Use Discriminative When:

  • Only need classification/regression
  • Have limited training data
  • Need fast training and inference
  • Want best accuracy for prediction
  • Don't need data generation
  • Building prediction systems

üí° Key Insights

The Fundamental Trade-off

Generative models solve a harder problem (modeling P(X,Y)) which gives them more capabilities but requires more resources.

Discriminative models solve an easier problem (modeling P(Y|X)) which makes them more efficient but less versatile.

‚ö†Ô∏è Common Misconception

Generative models aren't always worse at classification! With enough data, they can match discriminative models while offering additional capabilities like generation and outlier detection.

üß™ Test Your Understanding

Q1: Which type of model would you use to create deepfakes?

Generative Model (specifically GANs)
Deepfakes require generating new, realistic images/videos, which is only possible with generative models that understand how faces are constructed.

Q2: You're building a credit card fraud detector with limited data. Which approach is better?

Discriminative Model
With limited data, discriminative models (like Random Forest or XGBoost) will generally perform better as they focus only on the decision boundary rather than modeling the entire data distribution.

Q3: Can a generative model be used for classification?

Yes!
Generative models can classify by using Bayes' rule: P(Y|X) = P(X|Y)P(Y)/P(X). They learn P(X|Y) and P(Y), then compute P(Y|X) for classification.

üöÄ Cutting-Edge Hybrid Approaches

The Best of Both Worlds

Modern research explores hybrid approaches:

  • Semi-supervised learning: Use generative models to create synthetic training data, then train discriminative models
  • Adversarial training: GANs use both generative (generator) and discriminative (discriminator) components
  • Energy-based models: Bridge between generative and discriminative approaches
  • Flow-based models: Tractable generative models with discriminative performance
