Supervised Learning

Definition: Learning with a teacher! The algorithm learns from labeled training data.

üéØ Key Characteristic:
We know the right answer during training
üìä Input: Features + Labels
üìà Output: Predictions for new data
üîç

Unsupervised Learning

Definition: Learning without a teacher! The algorithm finds hidden patterns in unlabeled data.

üéØ Key Characteristic:

üìä Input: Features only (no labels)
üìà Output: Patterns, groups, or representations

‚öñÔ∏è Pros and Cons

Supervised Learning

‚úÖ Pros

  • Clear performance metrics
  • Predictable outputs
  • Well-defined objectives
  • Easy to evaluate

‚ùå Cons

  • Needs labeled data (expensive)
  • Can overfit to training data
  • Limited to known patterns
  • Biased by labels

Unsupervised Learning

‚úÖ Pros

  • No labeling required
  • Discovers hidden patterns
  • Works with raw data
  • Finds unknown insights

‚ùå Cons

  • Hard to evaluate results
  • Unpredictable outputs
  • Requires interpretation
  • No clear right answer

Python: Supervised Learning (House Prices)

from sklearn.linear_model import LinearRegression from sklearn.model_selection import train_test_split # Supervised: We have both features (X) and labels (y) X = [[2000, 3], [1500, 2], [2500, 4], [1800, 3]] # [size, bedrooms] y = [500000, 350000, 650000, 425000] # Known prices (LABELS!) # Split data into training and testing X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2) # Train the model with labeled data model = LinearRegression() model.fit(X_train, y_train) # Learning from examples # Predict on new house new_house = [[2200, 3]] predicted_price = model.predict(new_house) print(f"Predicted price: ${predicted_price[0]:,.0f}")

Python: Unsupervised Learning (Customer Clustering)

from sklearn.cluster import KMeans import numpy as np # Unsupervised: We only have features, NO LABELS! customers = np.array([ [25, 2000, 52], # [age, annual_spend, visits_per_year] [45, 8000, 12], [28, 2500, 48], [50, 9000, 10], [30, 3000, 45], [48, 7500, 15] ]) # Notice: No y labels! We don't know the "right" grouping # Let the algorithm find patterns kmeans = KMeans(n_clusters=2) kmeans.fit(customers) # Finding hidden structure # See which cluster each customer belongs to clusters = kmeans.predict(customers) print(f"Customer groupings discovered: {clusters}") # Output might be: [0, 1, 0, 1, 0, 1] # (Young frequent shoppers vs Older high spenders)

üéì Key Takeaways

Supervised = Teaching with Examples

  • Like teaching a child with flashcards
  • Show examples with correct answers
  • Test on new, similar problems
  • Measure success by accuracy

Unsupervised = Self-Discovery

  • Like organizing a messy room
  • Group similar things together
  • No "right" way to organize
  • Success = useful patterns found

üåç Combined Real-World Example: Amazon's Recommendation System

They use BOTH types of learning!

Supervised Component

Predicts if you'll buy a product based on:

  • Past purchases (labeled: bought/not bought)
  • Product ratings you gave
  • Items in cart

Unsupervised Component

Groups similar products and users:

  • Clusters products by features
  • Groups users with similar tastes
  • Finds buying patterns

Result: "Customers who bought this also bought..." (Unsupervised) + "Recommended for you" (Supervised)
